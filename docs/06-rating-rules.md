# 06 评分规则

## 文档用途

本文件定义 Agent Radar 如何给不同类型工具评分。评分结果必须可解释、可回归测试，并能帮助 AI 做选择。

## 应包含内容

- 通用评分维度：任务匹配度、维护活跃度、文档质量、安装复杂度、来源可信度、权限风险、社区信号。
- MCP 专属维度：权限范围、认证方式、协议兼容、工具描述质量、供应链风险。
- Skill 专属维度：触发条件清晰度、可移植性、指令质量、依赖资源、边界说明。
- Agent/Framework 专属维度：扩展性、生态、示例质量、运行成本、状态管理。
- CLI/SDK 专属维度：安装方式、API 稳定性、平台兼容、自动化友好性。
- 评分输出：总分、分项分、推荐等级、风险等级、解释。

## 后续生成提示词

```text
请为 Agent Radar 生成评分规则文档。要求不同工具类型使用不同评分维度：MCP、Skill、Agent、Framework、CLI、Prompt/Rules 分别定义评分项、权重、扣分项、加分项、风险项和解释模板。评分必须可复现、可测试、可被 AI 用于推荐决策。内容使用中文，并给出示例评分。
```

## 维护规则

- 修改评分规则必须同步更新评测集。
- 评分结果必须包含解释，不允许只有数字。
