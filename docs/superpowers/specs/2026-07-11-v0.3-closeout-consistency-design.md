# v0.3 收尾一致性清理设计

> 状态：已批准
>
> 实现提交：无
>
> 当前状态来源：[`docs/15-roadmap.md`](../../15-roadmap.md)

## 目标

清理 v0.3 已完成并发布后遗留的状态表述和本地生成 artifact，使权威文档一致指向 `all-v0.3.3`，并让工作区在不定义 v0.4 产品或视觉方案的前提下回到可继续开发的基线。

## 范围

- 修正 `docs/15-roadmap.md` 中仍把当前阶段描述为 v0.3 P1 的旧快照。
- 搜索 README 和 `docs/00-14`，确认不存在其他与“v0.3 已完成并发布”冲突的当前状态表述。
- 将本地 `public/data`、`public/reports`、`dist` 和 `dist-pages` 视为可再生、被 Git 忽略的构建输出；删除其中携带旧 release metadata 的 artifact，而不是手工改写真实 provider eval 结果。
- 保留并不触碰用户现有的 `src/ui/styles.css` 修改。
- 用文档一致性搜索、测试、lint、stylelint、production build 和 release check 验证清理结果；验证产生的可再生输出在最终交付前再次移除。

## 不在范围内

- 不创建 v0.4 Spec、Plan、视觉稿或实现。
- 不修改 v0.4 的产品范围、验收标准或优先级。
- 不重跑需要 provider 凭据的真实 LLM eval，不伪造 `all-v0.3.3` 的 eval metadata。
- 不修改或提交 `src/ui/styles.css`。

## 决策依据

`public/data/eval_summary.json` 和 `dist-pages/data/eval_summary.json` 不受 Git 跟踪，由 pipeline/release 构建生成。当前本地副本记录 `all-v0.3.3` 但携带较早的 commit SHA，说明它是开发机上的陈旧生成物，不是生产 evidence 的权威副本。生产事实由 Roadmap、`all-v0.3.3` tag 和 `production-release-evidence.json` 记录；因此安全的清理方式是删除可再生输出，而不是编辑评测结果。

## 验收标准

- Roadmap 当前进度明确为 v0.3 已完成并发布；v0.4 尚未开始设计。
- 当前状态搜索不再命中“当前开发阶段为 v0.3 P1”或“后续工作进入 v0.3 P1”。
- Git 状态除用户原有 `src/ui/styles.css` 外，只包含本次批准的文档变更。
- 247 项测试、ESLint、Stylelint、Vite production build 和 24/24 release check 通过。
- 最终工作区不保留 `dist`、`dist-pages`、`public/data` 或 `public/reports` 的陈旧生成输出。
