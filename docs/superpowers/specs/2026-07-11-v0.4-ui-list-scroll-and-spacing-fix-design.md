# v0.4 UI：列表滚动与间距修正

> 状态：已批准
>
> 实现提交：无
>
> 当前状态来源：[`docs/15-roadmap.md`](../../15-roadmap.md)、[`docs/14-web-ui.md`](../../14-web-ui.md)

## 问题

shadcn + Tailwind 重构后，顶部导航缺少视觉留白，filter hover 反馈不足，Tool list 长内容可能越界，Evaluation item 的 hover border 过于相邻；Tool 与 Evaluation 列表参与页面自然高度，记录较多时页面被显著拉长。

## 已批准方案

- 只为桌面端 Tool list 和 Evaluation list 设置基于 viewport 的 `max-height` 与 `overflow-y-auto`。
- 不修改详情栏定位或滚动方式，不引入 sticky detail、分页、虚拟列表或新依赖。
- 移动端保留 list → detail drill-in，不限制列表高度。
- 顶部导航增加上下视觉留白，并调整 nav variant，使选中与 hover 状态不贴住顶栏边缘。
- 提高 ToggleGroup filter 的 hover 对比度，选中状态仍由 shared primitive 管理。
- Tool row 增加完整的收缩、截断和固定分数宽度约束，防止内容越界。
- Evaluation list item 的垂直间距从紧邻提升为清晰分隔，避免相邻 hover border 接触。

## 验收

- Desktop Tool/Evaluation list 超过最大高度后在列表内部滚动。
- Desktop detail 仍按原页面流展示，不新增独立滚动或 sticky 行为。
- Mobile drill-in 行为不变，列表不被桌面 max-height 截断。
- Top navigation、filter hover、tool row overflow 和 Evaluation item spacing 均通过浏览器截图验证。
- `npm test`、`npm run lint`、`npm run stylelint`、`npm run pages:build` 全部通过。

## 不做

- 不修改推荐、评分、Evaluation 数据或反馈功能。
- 不新增 ScrollArea、分页或虚拟列表组件。
- 不调整详情栏信息架构。
