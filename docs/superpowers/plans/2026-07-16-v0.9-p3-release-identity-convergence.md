# v0.9 P3 Release Identity Convergence Implementation Plan

- Status: Approved
- Implementation commits: None
- Current status source: [docs/15-roadmap.md](../../15-roadmap.md)

**Goal:** Make deployed Worker and MCP release identity convergence a blocking, evidence-backed prerequisite for production release completion and Registry publication.

**Architecture:** Add one dependency-free release identity poller and CLI, reuse the canonical tag-to-version parser, make MCP smoke require the expected server version, bind both actual identity results into production evidence v2, and invoke the same gates from Release All and Publish MCP Registry.

## Constraints

- Do not duplicate convergence semantics in workflow shell.
- Do not accept copied workflow inputs as observed production identity.
- Do not expose response bodies or credentials in errors or evidence.
- Do not change the seven functional MCP smoke check IDs.
- Do not publish Registry metadata unless both production and fresh Registry checks match the selected tag/SHA/version.
