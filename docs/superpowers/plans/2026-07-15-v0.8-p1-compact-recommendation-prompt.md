# v0.8 P1 Compact Recommendation Prompt Implementation Plan

- Status: Completed
- Implementation commits: `8d8c06c6`
- Current status source: [docs/15-roadmap.md](../../15-roadmap.md)

**Goal:** Remove formatting-only whitespace from the provider recommendation prompt while preserving its parsed content and all recommendation and safety behavior.

**Architecture:** `buildRecommendationPrompt` remains the single prompt boundary. It constructs the existing instruction, result shape, query, and normalized catalog value, then emits compact JSON. Tests compare the serialized prompt with the compact serialization of its parsed value and freeze query plus catalog identity. Authoritative recommendation, evaluation, architecture, and Roadmap documents record the boundary and production verification requirement.

## Constraints

- Do not reorder prompt fields, remove catalog fields, change provider request messages, add retrieval, or change public schemas.
- Do not reduce Golden Queries, safety checks, or retries.
- Do not treat the MiniMax M3 cache probes as production evidence or persist their prompts/responses.
- Preserve unrelated worktree changes.
