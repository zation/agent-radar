# v0.6 Official MCP SDK and Registry Publication Implementation Plan

- Status: Approved
- Implementation commits: None
- Current status source: [docs/15-roadmap.md](../../15-roadmap.md)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the handwritten MCP adapter with the official TypeScript SDK v2, unify HTTP and MCP behavior, remove body credentials, and actually publish the verified production remote as `io.github.zation/agent-radar@0.6.0`.

**Architecture:** A transport-neutral Tool Contract defines four read-only tools and a Tool Service owns their business behavior. Thin HTTP and MCP adapters validate transport input, build request-scoped credential context, and call that service. A separate post-release workflow verifies immutable production evidence, publishes remote-only metadata through GitHub OIDC, queries the Registry, and records publication evidence.

**Tech Stack:** TypeScript, React, Cloudflare Workers, Node `node:test`, Zod 4.4.3, `@modelcontextprotocol/server@2.0.0-beta.3`, Wrangler, GitHub Actions, `mcp-publisher` v1.8.0.

## Global Constraints

- Pin the SDK and Zod exactly; use only the Web Standard server API.
- Keep the four public HTTP routes and stable business result shapes.
- Reject `api_key` in HTTP bodies and MCP arguments. Read `X-Agent-Radar-LLM-API-Key` into request-scoped memory only; optional environment fallback is `AGENT_RADAR_LLM_API_KEY`.
- Never log, persist, echo, checksum, or artifact the Provider key.
- Keep `server.json` remote-only and immutable for Registry version `0.6.0`.
- Do not mark v0.6 completed before production and Registry evidence both pass.
- Preserve unrelated worktree changes. Before every commit run focused tests and `git diff --check`.

- [ ] **Step 3: Register tools with the SDK**

Use this lifecycle and result shape:

```ts
import { createMcpHandler, McpServer } from "@modelcontextprotocol/server";

function createServer(service: ToolService, context: ToolRequestContext, version: string) {
  const server = new McpServer({ name: "io.github.zation/agent-radar", version });
  for (const name of MCP_TOOL_NAMES) {
    const contract = toolContracts[name];
    server.registerTool(name, {
      title: contract.title,
      description: contract.description,
      inputSchema: contract.input,
      outputSchema: contract.output,
      annotations: contract.annotations,
    }, async (input) => {
      try {
        const output = await service.execute(name, input, context);
        return { content: [{ type: "text", text: JSON.stringify(output) }], structuredContent: output };
      } catch (error) {
        const safe = toSafeToolError(error);
        return { isError: true, content: [{ type: "text", text: JSON.stringify(safe) }], structuredContent: safe };
      }
    });
  }
  return server;
}
```

Create a stateless handler per accepted POST whose factory closes over only that request's credential context. Confirm the pinned beta's exact type signature from installed declarations; adapt only the call signature.
