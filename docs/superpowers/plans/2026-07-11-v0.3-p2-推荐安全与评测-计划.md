# v0.3 P2：推荐安全与评测 Implementation Plan

> 状态：草稿
>
> 实现提交：无
>
> 当前状态来源：[`docs/15-roadmap.md`](../../15-roadmap.md)
>
> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为所有动态推荐入口增加确定性上下文安全评估、Recommendation Result v2、release/commit 溯源、24 条 golden queries 和 critical safety 发布门禁。

**Architecture:** 保留 LLM 的任务理解与候选选择职责，在候选归一化后调用独立的 `recommendation/safety.ts` 计算不可被模型解除的安全下限。Web、HTTP API、MCP 和 golden eval 共用同一个 Recommendation Result v2；Release All 只验证当前候选的 24 条真实 provider eval，不下载历史 artifact 或生成 Eval Diff。

**Tech Stack:** TypeScript、Node.js test runner、React/Vite、Cloudflare Worker、GitHub Actions、JSON reviewed bundle

## Global Constraints

- Recommendation Result 直接升级为 `recommendation_result.v2`，不提供 v1 adapter、版本协商或兼容输出。
- 持久化构建产物和其他机器接口继续保留各自的 `schema_version`。
- 安全门禁必须由确定性代码执行；LLM 只能更保守，不能解除 `ask_human`、`avoid` 或 `no_reliable_match`。
- Web 只展示“使用前确认事项”和安全默认值，不收集、保存或提交回答。
- Golden queries 总数固定为 24，其中 critical safety cases 固定为 4。
- 正式 Release All 中 critical case 失败、缺失、未执行、无 provider key 或 provider/schema error 均阻断发布。
- 不实现 Eval Diff、历史 artifact 下载、Prompt/provider/rules 独立版本号、审批状态或自动执行。
- 不修改 P1 Tool Card 采集、normalization、promotion 和数据可信度主链路。
