> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Translate all 24 Golden Query `query.task` and `review_notes` fields into English while proving that every non-text evaluation constraint and release-blocking safety rule remains unchanged.

**Architecture:** A committed JSON fixture freezes the protected projection of all 24 Eval Cases without the two translated fields or `updated_at`. Contract tests compare runtime data with that fixture and verify the Evaluation UI renders source text directly. The existing language validator gains structured Golden Query field scanning, and the existing provider-backed pipeline remains the behavioral release gate.

**Tech Stack:** TypeScript 5.5, Node.js standard library and test runner, JSON fixture, existing LLM-backed evaluation pipeline, Markdown.

## Global Constraints

- Modify only `query.task`, `review_notes`, and the uniformly refreshed `updated_at` inside the 24 Golden Queries.
- Preserve every case ID, count, category, severity, owner, expected value, risk tolerance, permission, environment, tool type, tag, and stack constraint.
- Keep exactly 24 cases, 24 unique IDs, and four critical cases.
- Do not add UI translations, runtime i18n, bilingual artifacts, dependencies, or safety-gate exceptions.
- The language rule applies only to each Golden Query's `query.task` and `review_notes`; it does not scan unrelated TypeScript comments or external evidence.
- A missing provider key or any provider, schema, or quality failure cannot satisfy P2 completion.
- Use `2026-07-13T00:00:00Z` as the uniform P2 `updated_at` value.
