**Goal:** 把当前多页面 shadcn 风格 UI 重构为包含统一 Tools 决策工作区和公开 Evaluation 页面的浅色 AI 决策终端，并完整实现推荐状态与移动端 drill-in 行为。

**Architecture:** 保留 React/Vite、现有推荐 API 和发布 artifacts。`App.tsx` 只负责加载数据与组合页面；推荐工作区、Tool 详情、Evaluation 和移动端导航拆为独立组件，跨组件决策由纯 view-model 函数承担并使用 Node tests 锁定。`golden_queries.json` 与 `eval_summary.json` 在 UI 数据层合并，不在浏览器重算评测。

**Tech Stack:** TypeScript 5.5、React 19、Vite 8、Tailwind CSS 4、Base UI/shadcn primitives、Lucide React、Node test runner、Stylelint、ESLint。

## Global Constraints

- 一级导航只显示 `Tools` 和 `Evaluation`；不迁移或重新设计 Compare、Review。
- Recommend 成功后直接重排 Tools 列表，不渲染独立结果卡片或说明面板。
- Recommend 首次/无任务展开，成功后折叠，loading、`ask_human`、`no_reliable_match` 和 error 保持展开。
- Success 只显示 check icon、任务摘要和 `N candidates ranked`；其他状态只在输入框下显示单行信息。
- 状态、选中和强调必须使用不同视觉语义。
- `Evaluation` 使用现有 `golden_queries.json` 和 `eval_summary.json`，不在浏览器重跑评测。
- 移动端 Tools 与 Evaluation 都使用 list → detail drill-in，并恢复筛选和滚动位置。
- 不引入新运行时依赖、深色模式、营销首页、装饰渐变或默认 shadcn Card 网格。
- 本 Plan 不实现 GitHub OAuth、D1 投票或 Issue Form 写链路；反馈控件的持久化与弹窗集成进入独立 Plan，不能用本地假状态伪装成功。

- [ ] **Step 4: Implement the pure state model**

Add:

```ts
export type RecommendationUiKind = "idle" | "loading" | "success" | "ask_human" | "no_reliable_match" | "error";

export interface RecommendationUiState {
  kind: RecommendationUiKind;
  shouldCollapse: boolean;
  inlineMessage?: string;
}

export function getRecommendationUiState(input: {
  isSubmitting: boolean;
  result: RecommendationResult | null;
  error: string;
}): RecommendationUiState {
  if (input.isSubmitting) return { kind: "loading", shouldCollapse: false };
  if (input.error) return { kind: "error", shouldCollapse: false, inlineMessage: input.error };
  if (!input.result) return { kind: "idle", shouldCollapse: false };
  if (input.result.recommended_action === "ask_human") {
    return { kind: "ask_human", shouldCollapse: false, inlineMessage: input.result.safety_assessment.approval_reason ?? "Human confirmation is required for this task." };
  }
  if (input.result.recommended_action === "no_reliable_match") {
    return { kind: "no_reliable_match", shouldCollapse: false, inlineMessage: input.result.no_match_reason ?? "No reliable match in the reviewed catalog." };
  }
  return { kind: "success", shouldCollapse: true };
}
```
