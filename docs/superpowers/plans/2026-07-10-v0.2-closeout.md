# v0.2 收尾实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 完成并发布 `all-v0.2.5`，使审核语义、生成产物、文档、质量门禁、production evidence 和线上 Worker 状态一致。

**Architecture:** 保留现有 ingestion、auto review、release admission、promotion check 和单 Worker 部署架构，只修正仍带有逐条人工审核暗示的 artifact 字段与文案。新增一个独立、可单元测试的 production evidence builder，由 GitHub Actions 在 production 人工审批、部署和 MCP smoke 之后调用，将 approved run、deployment、bundle checksums 和线上 endpoint 固化为 JSON artifact。

**Tech Stack:** TypeScript ES2022、Node.js 22+、Node test runner、React/Vite、ESLint、Stylelint、GitHub Actions、`gh` CLI、Cloudflare Wrangler。

## Global Constraints

- 审核本身由脚本、规则和 LLM 完成，正常人工确认只发生在 GitHub `production` environment gate。
- `approval_record.v1` 只作为 break-glass `approval_override`，不生成逐条人工 approval 表单或 JSON template。
- 推荐阶段高风险工具的 `requires_human_approval` 继续保留，不得与发布审核语义混淆。
- Provider 配置 UI 和 browser direct-to-provider 模式不进入 v0.2。
- 不修改评分权重、不扩大来源范围、不启用 D1 serving、不新增依赖。
- Web、artifacts、HTTP API 和 MCP 继续部署在同一个 Cloudflare Worker 中。
- GitHub 操作统一使用 `gh` CLI；production 审批仍由维护者在 GitHub 手动完成。

```ts
export interface ProductionReleaseEvidence {
  schema_version: "production_release_evidence.v1";
  github: { repository: string; run_id: string; sha: string; release_tag: string };
  deployment: { id: string; environment: "production"; worker_base_url: string; mcp_endpoint: string };
  bundle: { artifact_name: string; manifest_sha256: string; d1_seed_sha256: string };
  smoke: { passed: true; total: number; passed_checks: number; failed: 0 };
  generated_at: string;
}

export async function buildProductionReleaseEvidence(
  options: BuildProductionReleaseEvidenceOptions
): Promise<ProductionReleaseEvidence>;

export function renderProductionReleaseEvidenceMarkdown(
  evidence: ProductionReleaseEvidence
): string;
```

使用 `node:crypto` 对 manifest 和 D1 seed 计算 `sha256:<hex>`；读取 `ArtifactManifest` 与 `McpSmokeResult`，逐项执行设计文档中的失败校验。
