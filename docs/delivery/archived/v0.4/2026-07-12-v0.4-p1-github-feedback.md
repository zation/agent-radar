**Goal:** 在单 Cloudflare Worker 中实现 GitHub OAuth、签名身份 Cookie、D1 Tool 投票、Tool Detail thumb 反馈和 GitHub Issue Form 入口。

**Architecture:** `src/feedback/*` 提供身份、OAuth、反馈路由和 D1 adapter，`src/worker.ts` 在现有 artifact API 前分流 auth/feedback endpoints。React 使用独立 feedback client/provider 管理身份、聚合和 optimistic mutation；P1 不修改 Rating，也不读取或处理 GitHub Issues。

**Tech Stack:** TypeScript 5.5、Cloudflare Workers、D1 SQLite、Web Crypto、React 19、shadcn/Base UI、Tailwind CSS 4、Node test runner、Vite 8。

## Global Constraints

- OAuth 不发送 scope，只读取 GitHub public `id/login`。
- Token、OAuth code、Cookie 和 secret 不进入 D1、artifact、响应或日志。
- Session 为 30 天 HMAC HttpOnly Cookie；OAuth state 为 10 分钟签名 Cookie。
- 同一 `(github_user_id, tool_id)` 最多一条当前 `up`/`down` 投票。
- 匿名可读聚合；写入校验 session、Origin、JSON、Tool ID 和 30/min/user 限流。
- Worker/UI 不直接创建 Issue；P1 不实时调整 Rating、风险或 trust。
- 现有 HTTP API、MCP、Static Assets、Recommend 和 Evaluation 保持兼容。

## File Structure

- `migrations/0002_feedback_votes.sql`：vote 与 mutation rate-limit schema。
- `src/feedback/contracts.ts`：身份、vote、aggregate 类型。
- `src/feedback/cookies.ts`：HMAC session/state Cookie。
- `src/feedback/github-oauth.ts`：GitHub OAuth transport。
- `src/feedback/store.ts`：D1 feedback adapter。
- `src/feedback/http.ts`：auth/feedback router。
- `src/ui/feedback-client.ts`、`feedback-state.ts`、`feedback-provider.tsx`：browser state。
- `src/ui/tool-feedback.tsx`、`identity-menu.tsx`：UI。
- `.github/ISSUE_TEMPLATE/tool-feedback.yml`：结构化反馈表单。
