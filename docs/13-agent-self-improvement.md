# 13 Agent 自迭代机制

## 文档用途

本文件定义 coding agent 如何在 Agent Radar 项目中自行发现问题、提出改进、修改代码或数据，并通过评测和人类审批安全迭代。

## 应包含内容

- 自迭代触发：采集失败、解析失败、推荐误判、评分异常、schema 缺口、用户反馈。
- 可自动处理任务：低风险 parser 修复、数据补全、文档更新、评测样例增加。
- 需要审批任务：schema 语义变化、评分大改、删除数据、新增高风险来源、自动安装工具。
- 改进流程：发现 -> issue/task -> 分支 -> 修改 -> 测试 -> eval diff -> 变更报告 -> 审核。
- Eval diff 格式：变更前后推荐结果、评分变化、风险变化、失败案例。

## 后续生成提示词

```text
请为 Agent Radar 生成 Agent 自迭代机制文档。请定义 coding agent 可以如何根据采集失败、数据质量问题、推荐误判和评测失败自动提出改进。要求明确允许自动修改的范围、必须人类审批的范围、任务生成格式、PR/变更报告格式、评测前后对比和安全护栏。内容使用中文。
```

## 维护规则

- 自迭代不是无监督自我修改，所有高风险变更必须有人类或明确策略审批。
- Agent 的每次改动都应能被回放、解释和回滚。
