# 01 需求文档

## 文档用途

本文件记录 Agent Radar 的功能需求、非功能需求、MVP 范围和延期范围。它用于指导开发计划和验收标准。

## 应包含内容

- 功能需求：采集、标准化、分类、评分、搜索、推荐、MCP 查询、Web UI、报告生成。
- 非功能需求：可维护性、可扩展性、成本、性能、数据新鲜度、可解释性、安全性。
- MVP 范围：优先覆盖 MCP、Skill、Agent/CLI 工具；先支持静态数据和轻量 MCP 查询。
- 延期范围：复杂账号系统、在线安装市场、企业权限治理、高级可视化。
- 验收标准：每项能力对应输入、输出、质量标准和测试方式。

## 后续生成提示词

```text
请为 Agent Radar 生成一份完整需求文档。请基于产品目标拆分功能需求和非功能需求，明确 MVP 必须实现、可以延期、不应该做的内容。每条需求都要包含验收标准。重点覆盖工具采集、Tool Card 标准化、分类、评分、推荐、AI 友好输出、MCP 查询接口、评测和安全护栏。内容使用中文。
```

## 维护规则

- 新增需求必须说明目标用户和验收方式。
- 不要把“未来可能有用”的想法直接塞入 MVP。
